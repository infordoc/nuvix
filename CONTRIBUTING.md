## How to Contribute

### 1. Reporting Issues
- Search existing issues before opening a new one.
- Provide as much detail as possible (steps to reproduce, expected behavior, environment details).

### 2. Submitting Changes
1. Fork the repository.
2. Create a new branch for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. Make your changes with clear and concise commit messages.
4. Push your branch and open a pull request.

### 3. Coding Guidelines

* Use **TypeScript** consistently.
* Follow existing code style and conventions.
* Write unit tests for new features and bug fixes.
* Keep pull requests focused and minimal.

### 4. Communication

* Use GitHub Discussions for general questions.
* Use Issues for bugs and feature requests.
* Join in on code reviews and provide constructive feedback.

## Code of Conduct

We are committed to fostering a welcoming and inclusive environment.

## Getting Help

If you need help setting up or contributing, please open a discussion or reach out via issues.
